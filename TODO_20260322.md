## A. 管線問題（影響數據品質）

### A1. [!!!] R/B 通道比較星選取失敗 — 星表欄位缺失（非飽和問題）
- **現象**：V1162Ori R 通道最終通過 0 顆比較星
- **影響**：R/B 通道完全無法跑；未來要測參宿四（V~0.5）更是無望
- **根因（2026-03-22 更新）**：
  **不是飽和排除**，而是 `_pick_col()` 找不到對應波段星等欄位：
  - R 通道 → `phot_band="r"` → 找 `rmag/mag_r` → 星表 CSV 只有 `vmag` → KeyError
  - APASS 原始 VizieR 查詢有多色數據，但 `catalog_APASS.csv` 保存時只留 `vmag`
  - 飽和閾值 65536 在實際觀測中幾乎不會觸發（最亮星 peak ~49000）
- **需要做的事**：
  1. ✅ 已完成：m_inst vs m_cat 線性度圖（見 FINDINGS §5）
  2. 修改星表保存邏輯：保留 APASS 原始多色星等（bmag, vmag, rmag, imag）
  3. 或修改 R/B 通道的 fallback：找不到對應波段時改用 vmag + 色項修正
  4. 確認 B 通道是否有同樣問題（CC And B 通道曾跑成功，需確認是 AAVSO 有 B 星等還是另有原因）

### A2. Gaia301 新結果比較星數驟降（60 -> 5 顆）
- **現象**：重跑後 comp_used 從 60 降到 3-5，ZP slope=1 fallback
- **影響**：m_var std 從 0.002 變成 0.36（不可用）
- **根因**：待查，可能是 AAVSO/APASS 星表查詢返回不同結果，或緩存失效
- **需要做的事**：比對新舊 run 的 catalog CSV（在 1_photometry/catalogs/ 下）

### A3. V1373Ori 比較星選取失敗
- **現象**：亮度範圍篩選後通過 0 顆
- **影響**：完全無法測光
- **根因**：V1373Ori 亮度 8.72V，comp_mag_range 可能太窄
